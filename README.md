Hello,

In my work, I often think about AI governance, experiment with participative context and decision architectures, and try to measure how those affect both the systems we design and the organisations we design them for. I call this line of inquiry institutional cognition for short – not in the sense of cognisant institutions, but by considering the cognitive throughput an organisation collectively processes, how little of it minimally needs to meaningfully function, and how much it can maximally absorb before it implodes.

Full disclosure: I've held several tech roles in my career, and all of them were on the project or team leadership spectrum. I'm by no means a fluent coder and respect coders and their skills too much to imply otherwise. My contributions so far live mostly in thoughtful Jira tickets, not in pull requests. I'm using coding agents in almost all of the work I publish here. I know that agentic code creates its own class of downstream problems, so I fend off slop as much as I can. But here's the thing: the code isn't actually so much the point I want to make.

The ambition for this work is to be a conversation starter, or perhaps an emergent practice community: a place to think about human agency in agentic environments, propose safe and thoughtful conditions under which human and non-human intelligence generate value in partnership instead of randomly excavating it from 10X hype cycles.

So, why the code, if I'm mostly here to talk? Because code is a shared surface that keeps a conversation honest. Choosing it as the lens was both intentional and pragmatic: the code already functionally mirrors the community aspiration — make shared truth an emergent property of the work that surrounds it, give everyone equal access to it, and invite scrutiny and falsification at every step.

If you want to see what that looks like in practice, the bench currently holds:

- [context-mediation-layer](https://github.com/sebb001/context-mediation-layer) – a working prototype of the coordination layer itself
- [cml-evidence-pack](https://github.com/sebb001/cml-evidence-pack) – six governed cases from the lab where it runs, records included
- [epilab](https://github.com/sebb001/epilab) – an agent-based laboratory that stress-tests the governance primitives
- [Governing Epistemic Systems](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=6417898) – the working paper the simulations feed
- [essays on Substack](https://sebbln1.substack.com) – the longer arc, in prose
